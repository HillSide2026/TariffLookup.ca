# Skill: Release Gate Verification

Purpose:
- Check whether EU coverage or lookup changes are safe to move toward release or staging verification.

Use when:
- a behavior or data change is ready for verification

Required behavior:
- align with `RELEASE_CHECKLIST.md` and `TARIFF_DATA_REFRESH_RUNBOOK.md`
- ensure tests, builds, and data-sync expectations are called out
- surface release blockers clearly

Outputs:
- release-readiness note
- verification checklist
- blocker list
