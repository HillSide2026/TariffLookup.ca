# Skill: Negative Case Design

Purpose:
- Create tests and logic guards for things the system must not classify or normalize incorrectly.

Use when:
- neighboring categories collide
- exclusions and blocked logic need proof

Required behavior:
- identify adjacent product types
- add explicit non-match or non-normalize coverage
- use concrete examples, not abstract placeholders

Outputs:
- negative test case
- exclusion proof
- misclassification guard
