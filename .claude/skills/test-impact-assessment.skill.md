# Skill: Test Impact Assessment

Purpose:
- Decide which tests must change when EU logic, data, or guidance changes.

Use when:
- a code or dataset change affects lookup behavior

Required behavior:
- identify impacted backend, frontend, and normalization tests
- note whether coverage exists already or must be added
- avoid shipping behavior changes without test consideration

Outputs:
- impacted-test list
- coverage gap note
- recommended verification path
