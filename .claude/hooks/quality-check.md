# Quality Check (Operational)

Checklist (must pass before release or promotion):
- Material claims are supported by real repo files such as `README.md`, `docs/product/*`, `docs/engineering/*`, or implemented frontend files.
- Output clearly distinguishes implemented behavior from planned work, prototype behavior, or external handoff work.
- Marketing and UI language matches `docs/brand/TARIFFLOOKUP_BRAND_SYSTEM.md`.
- No unsupported claims about tariff certainty, legal advice, pricing, jurisdiction coverage, or launch status.
- Any external-facing output is still treated as draft until ML1 approves it.
- Any task requiring Canva, Figma, WordPress, Thrive, or a live deployment is described as a human-executed next step unless the repo actually contains the change.
- Findings, revisions, or blockers are explicit rather than silently rewritten away.
